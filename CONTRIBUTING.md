# Contributing

## Setup

```bash
git clone https://github.com/anthoadc-AI/chask-plant-modernization.git
cd chask-plant-modernization
make install
```

## Workflow

- Branch from `main`. Use conventional commit prefixes: `feat:`, `fix:`, `docs:`, `chore:`, `test:`.
- Before opening a PR, run `make lint` and `make test` — both must pass.
- Use `make format` to auto-fix lint and formatting issues.

## Analytical honesty

Any figure added to code, notebooks, or documentation must match the real dataset
(`data/raw/monthly_reconstructed.csv`). Synthetic data must be generated by a
reproducible script in `src/chask/datagen/` with a fixed seed, and explicitly
labeled as synthetic in filenames and docs.

## Code style

- Python >= 3.10, type hints on all public functions, Google-style docstrings in English.
- Line length: 100. Ruff rules: E, F, I, N, UP.
- No comments that describe *what* the code does — only *why* when non-obvious.
