# CLAUDE.md — chask-plant-modernization

Persistent context and rules for Claude Code. Read this before any task.

## Project overview

Portfolio-grade repository documenting a real consulting project: the plant
modernization and energy optimization of **Panificadora Chask**, an industrial
bakery in Punata, Cochabamba, Bolivia (Dec 21, 2020 – Jun 4, 2022), executed by
**INGEDAV S.R.L.** with **Anthony Davila** as project director. The repo
demonstrates competency in project management, energy efficiency, process
optimization, data engineering, and data science.

- GitHub: `anthoadc-AI/chask-plant-modernization`
- Intervention cutoff: **August 2021** (new machinery fully operational)
- Real dataset: 29 monthly observations, Jan 2020 – May 2022
  (Pre: n=20, Jan 2020–Aug 2021 · Post: n=9, Sep 2021–May 2022)
- Total project investment: USD 85,000

## Golden rule: analytical honesty

**Every figure in reports, docs, README, and dashboard must match what the data
actually shows.** Ground truth from `data/raw/monthly_reconstructed.csv`
(reproduced by `src/chask/datagen/monthly_reconstruction.py`, seed=42):

| Metric | Pre (n=20) → Post (n=9) | Direction |
|---|---|---|
| Energy consumption (kWh) | 51,827 → 41,689 | **-19.6%** improved |
| Energy intensity (kWh/kg) | 3.81 → 2.76 | **-27.5%** improved |
| Gross margin | 21.4% → 29.0% | **+7.5 pp** improved |
| Sales (USD) | 20,756 → 23,100 | **+11.3%** improved |
| Machine failures /month | 8.1 → 4.3 | **-46.2%** improved |
| Downtime (h/month) | 26.9 → 15.7 | **-41.7%** improved |
| Production (kg) | 13,680 → 15,249 | **+11.5%** improved |

**Honest framing rules:**
- All 7 headline metrics improve in the post period vs. pre.
- Sep–Oct 2021 show a commissioning spike (failures: 10, 9 vs pre mean 8.1)
  that is visible in figures and must be noted in any report referencing failures.
- The post-period average includes that spike; steady-state (Dec 2021–May 2022)
  is even better: energy ~40,062 kWh/month (**-26.3% vs 54,388 kWh baseline**).
- The engineering report's steady-state claims (-27% energy, +50% capacity,
  +22% productivity) align with the steady-state slice of the dataset, not the
  full post-period average; always distinguish the source when citing figures.
- The monthly dataset is a **documented reconstruction** (original records are
  confidential); disclose this whenever citing figures. See data dictionary.

## Synthetic data policy

- Synthetic/augmented datasets are allowed and encouraged, but must be:
  1. Generated by a reproducible script (`src/chask/datagen/`) with a fixed seed.
  2. Calibrated against the real monthly aggregates.
  3. Explicitly labeled as synthetic in filenames, docs, and data dictionary
     (e.g. `daily_synthetic.csv`).
- Never present synthetic data as real measurements.

## Code conventions

- Python >= 3.10. `src/` layout, package name `chask`, `pyproject.toml` packaging.
- Lint/format: run `ruff check --fix` **and** `ruff format` as two separate
  steps (formatting is not covered by check).
- scikit-learn matrix variables named `X` require `# noqa: N806`.
- Prefix intentionally unused variables with `_`.
- Every module in `src/chask/` gets pytest unit tests in `tests/`. CI must stay
  green on Python 3.10, 3.11, and 3.12.
- Type hints on all public functions. Docstrings in English (Google style).
- Notebooks delegate logic to `src/chask/`; no heavy logic inside notebooks.
- Streamlit and MkDocs (later phases) also delegate computation to the package.

## Repository language

- Code, docstrings, README, docs site: **English** (international audience).
- Source documents from the client engagement remain in Spanish under
  `docs/source-documents/` and are referenced, not translated wholesale.

## Git workflow

- Branch: `main`. Semantic, real commits (conventional commits style:
  `feat:`, `fix:`, `docs:`, `chore:`, `test:`).
- One tag per completed phase: Fase 0 → `v0.1.0`, Fase 1 → `v0.2.0`,
  Fase 2 → `v0.3.0`, Fase 3 → `v0.4.0`, Fase 4 → `v0.5.0`, Fase 5 → `v1.0.0`.
- **Never** create artificial history (no backdated commits, no history
  scripts). Never commit template/boilerplate artifacts without reviewing them.
- Hardcoded URLs (README badges, `pyproject.toml`, `mkdocs.yml`) must all point
  to `anthoadc-AI/chask-plant-modernization` consistently.

## Phase plan (approved)

0. Foundations & governance — repo skeleton, tooling, CI, project-management docs
1. Data engineering — raw/staging/analytics pipeline, schema validation,
   data dictionary, synthetic daily dataset
2. Data science — EDA, anomaly detection (Z-score + Isolation Forest),
   hypothesis testing, interrupted time series / structural break analysis
3. Energy efficiency & process optimization — energy KPIs, motor replacement
   savings model, scenario-based ROI
4. Visible products — Streamlit dashboard, MkDocs Material site (incl. PM section)
5. Portfolio polish — README, CASE_STUDY.md (English), release v1.0.0

Work strictly within the scope of the phase requested. At the end of each
phase: run `make lint test`, report results, commit, tag, and stop for review.
