    Data Dictionary — Panificadora Chask

    Project: Plant Expansion and Resource Optimization
    Package: chask v0.3.0
    Last updated: 2026-07-01

    1. Raw Monthly Dataset

    File: data/raw/monthly_reconstructed.csv
    Generated by: src/chask/datagen/monthly_reconstruction.py (seed = 42)
    Source disclosure: The original paper records from the Panificadora Chask engagement are confidential client property. This file is a documented reconstruction generated from the engineering report metrics (pre-/post-intervention averages, steady-state measurements, and known anomaly months). It is reproducible from source with a fixed seed and calibrated to match the documented project outcomes. See § "Data Reconstruction Disclosure" below.
    Rows: 29 (Jan 2020 – May 2022)

    Column Type Unit Description Source
    fecha date — Last calendar day of the month (e.g., 2020-01-31). Unique, end-of-month, no gaps. Monthly record
    produccion_kg float kg Total bakery production for the month. Production log
    consumo_kwh float kWh Total electrical energy consumed by the plant. EMS / electricity bill
    fallas_maquina int count Number of machine failure events during the month. Maintenance log
    mantenimiento int count Number of planned maintenance interventions. Maintenance log
    ventas_usd float USD Total monthly revenue from product sales. Sales records
    costos_usd float USD Total monthly operating costs (energy + labor + materials). Cost records
    tiempo_inactividad_horas float hours Total production downtime (planned + unplanned) per month. Operations log

    2. Staging Dataset

    File: data/staging/monthly_validated.csv
    Generated by: chask.pipeline.transform.to_staging
    Description: The raw dataset after passing Pandera schema validation. Columns and values are identical to the raw dataset; types are coerced to their canonical dtypes. No new columns are added at this layer.

    Column Type Unit Description
    (same as raw) — — All raw columns, validated and type-coerced.

    Validation rules (enforced by chask.pipeline.validate.MONTHLY_SCHEMA):

    • fecha: datetime, unique, end-of-month, continuous monthly series (no gaps), 29 rows.
    • produccion_kg: float > 0.
    • consumo_kwh: float > 0.
    • fallas_maquina: int ≥ 0.
    • mantenimiento: int ≥ 0.
    • ventas_usd: float > 0.
    • costos_usd: float > 0.
    • tiempo_inactividad_horas: float ≥ 0.

    3. Analytics — Monthly Enriched Dataset

    File: data/analytics/monthly_enriched.csv
    Generated by: chask.pipeline.transform.to_analytics
    Description: Staging dataset enriched with derived KPI features.

    Column Type Unit Description Derivation
    (all staging columns) — — Inherited from staging. —
    intensity_kwh_kg float kWh/kg Energy intensity: energy consumed per kg of production. consumo_kwh / produccion_kg
    gross_margin_pct float % Gross margin as a percentage of revenue. (ventas_usd − costos_usd) / ventas_usd × 100
    profit_usd float USD Absolute gross profit. ventas_usd − costos_usd
    cost_per_kg float USD/kg Operating cost per kg of production. costos_usd / produccion_kg
    period str — Pre/post intervention label. "pre" if fecha ≤ 2021-08-31, else "post". Derived from INTERVENTION_CUTOFF

    Pre/post split:

    Period Rows Date range
    pre 20 Jan 2020 – Aug 2021
    post 9 Sep 2021 – May 2022

    Synthetic Data Disclosure

    This dataset is entirely synthetic and must not be used as real measurements.

    What is synthetic

    All rows in daily_synthetic.csv are computer-generated. No daily operational data was collected during the project; only monthly aggregates were recorded.

    How it was generated

    The generator (src/chask/datagen/synthetic.py) distributes each monthly aggregate across the calendar days of that month using:

    1. Weekday seasonality: relative production weights per day of week (higher on Friday/Saturday, lower on Sunday) reflecting typical bakery operations.
    2. Multiplicative lognormal noise (σ = 0.15) for day-to-day variability.
    3. Rescaling: each daily weight vector is rescaled so the sum exactly matches the monthly total.
    4. Failure events: fallas_maquina and mantenimiento are distributed as binary daily events (0 or 1) sampled without replacement, so the monthly count is preserved exactly.
    5. Consumption on failure days: failure days receive a 30% upward weight for consumo_kwh before rescaling, reflecting wasted energy during unplanned stops.

    Calibration guarantee

    For every calendar month, the sum of daily values equals the corresponding monthly value:

    • Continuous variables (produccion_kg, consumo_kwh, ventas_usd, costos_usd, tiempo_inactividad_horas): exact within floating-point precision (rtol ≤ 1e-9).
    • Discrete events (fallas_maquina, mantenimiento): exact integer match.

    This is verified by the test suite (tests/test_synthetic.py::test_calibration_*).

    Seed

    Fixed seed: RANDOM_SEED = 42 (defined in chask.config). Two runs with the same seed produce identical output (verified by tests/test_synthetic.py::test_reproducibility).

    Intended use

    The synthetic daily dataset is intended for:

    • Visualizations requiring sub-monthly granularity (time-series plots, Streamlit dashboard).
    • Anomaly detection algorithm development and testing (Z-score, Isolation Forest).
    • Demonstration and educational purposes.

    It is not suitable for: precise financial reporting, regulatory submissions, or any analysis where the accuracy of individual daily measurements is required.

    Data Reconstruction Disclosure

    The monthly dataset (monthly_reconstructed.csv) was generated computationally because the original operational records are confidential client property. The reconstruction is:

    1. Reproducible: generated by src/chask/datagen/monthly_reconstruction.py with a fixed RANDOM_SEED = 42. Running the script produces an identical file.
    2. Calibrated: target values for each month were derived from the engineering report, which documents pre-/post-intervention averages, steady-state measurements, and specific anomaly months (equipment failures in Apr 2020, Oct 2020, Mar 2021; commissioning spike in Sep–Oct 2021).
    3. Labeled: the filename, data dictionary, README, and all analysis outputs identify this dataset as a reconstruction. It is never described as raw operational data.

    Ground truth (pre/post means from the reconstructed dataset):

    Metric Pre (n=20) Post (n=9) Change Direction
    Energy consumption (kWh) 51,827 41,689 −19.6% improved
    Energy intensity (kWh/kg) 3.81 2.76 −27.5% improved
    Gross margin (%) 21.4 29.0 +7.5 pp improved
    Sales (USD) 20,756 23,100 +11.3% improved
    Machine failures /month 8.1 4.3 −46.2% improved
    Downtime (h/month) 26.9 15.7 −41.7% improved
    Production (kg) 13,680 15,249 +11.5% improved

    Steady-state slice (Dec 2021 – May 2022): energy ≈ 40,062 kWh/month (−26.3% vs 54,388 kWh baseline), consistent with the field-documented −27% target.

    Commissioning spike (Sep–Oct 2021): machine failures peak at 10 and 9 per month before returning to ≤3 in steady state. This spike is visible in all time-series figures and must be disclosed when interpreting post-period failure counts.

    Known Data Caveats

    1. Small post-intervention sample (n = 9)

    The post-intervention period covers only 9 months (Sep 2021 – May 2022). Statistical tests on this period have limited power. Effect size (Cohen's d) should always be reported alongside p-values to assess practical significance.

    2. Commissioning spike in Sep–Oct 2021

    The new machinery was commissioned in August 2021. The first two post-intervention months show elevated machine failures (10 and 9/month vs. steady-state 2–3/month). These are transient commissioning effects, not permanent regressions, and are visible in all figures.

    3. Distinction between dataset averages and steady-state field measurements

    The engineering closure report documents steady-state measurements after full calibration:

    • Energy reduction: −27% (55,000 → 40,000 kWh/month).
    • Capacity increase: +50%.
    • Productivity improvement: +22%.

    These figures were measured in controlled conditions and align with the Dec 2021–May 2022 steady-state slice of the dataset. When citing results, always specify whether you are quoting monthly dataset averages or field-documented steady-state measurements.

    4. No daily raw data

    No daily operational data was collected. The daily synthetic dataset is a model-generated approximation, not a record of actual daily operations.
